# Face Verification — Triplet Loss Model

A face verification system built with a Siamese ResNet-50 backbone trained using triplet loss. Given two face images, the model outputs a similarity score and decides whether they belong to the same person.

---

## Technologies

| Component | Choice | Reason |
|---|---|---|
| Framework | PyTorch 2.6.0 | Dynamic graph, best research support |
| Backbone | ResNet-50 | Strong pretrained ImageNet features, well proven for face tasks |
| Loss | Triplet Loss (α=0.2) | Metric learning — pulls same-identity embeddings together |
| Mining | Semi-hard negatives | Stable training, avoids gradient collapse |
| Embedding | 128-d L2-normalised vector | Unit sphere — cosine similarity = dot product |
| Learning method | One-shot / Siamese | Single network, shared weights across all three branches |
| Optimiser | Adam + Cosine LR decay | Fast convergence, smooth annealing |
| Augmentation | Flip + ColorJitter | Light augmentation appropriate for face data |

## Model Architecture

```
Input image (3 × 112 × 112)
        │
   ResNet-50 backbone          ← pretrained on ImageNet, FC layer removed
        │
   2048-d feature vector
        │
   Linear(2048 → 512)
   BatchNorm1d(512)
   ReLU
   Dropout(0.3)
        │
   Linear(512 → 128)
        │
   L2-Normalise                ← output lives on the unit sphere
        │
   128-d embedding vector
```

The **same network** processes Anchor, Positive, and Negative — this is the Siamese weight-sharing property.

---

## Triplet Loss

```
L = max(0,  d(A, P)  −  d(A, N)  +  α)

where:
  A   = Anchor embedding
  P   = Positive embedding (same identity as A)
  N   = Negative embedding (different identity)
  d   = squared L2 distance (= 2 − 2·cosine_similarity for unit vectors)
  α   = margin = 0.2
```

**Semi-hard mining** selects negatives where:
```
d(A, P)  <  d(A, N)  <  d(A, P) + α
```

## How to Run

### Step 1 — Generate splits (run once)

```bash
python split_dataset.py
```

Output: `splits/train.txt`, `splits/val.txt`, `splits/test.txt`

### Step 2 — Smoke tests (optional but recommended)

```bash
python dataset.py    # verifies image loading and tensor shapes
python model.py      # verifies ResNet-50 output and L2 norms
python loss.py       # verifies triplet loss computation
```

### Step 3 — Train

```bash
python train.py --workers 2
```

**Common flags:**

| Flag | Default | Description |
|---|---|---|
| `--epochs` | 30 | Number of training epochs |
| `--batch_size` | auto (4 for MX350) | Batch size |
| `--lr` | 1e-4 | Learning rate |
| `--margin` | 0.2 | Triplet loss margin |
| `--workers` | 4 | DataLoader worker threads |
| `--resume` | None | Path to checkpoint to resume from |

**Resume after interruption:**
```bash
python train.py --resume checkpoints/last_model.pt --workers 2
```

### Step 4 — Verify two faces

```bash
python verify.py --img1 path/to/face1.ppm --img2 path/to/face2.ppm
```

Output:
```
Similarity : 0.8731   (threshold=0.90)
Distance   : 0.2538
Decision   : SAME PERSON
```

## Metrics

### During training (logged per epoch to `checkpoints/training_log.json`)

| Metric | Description | Good direction |
|---|---|---|
| `loss` | Mean triplet loss | ↓ decreasing |
| `pos_dist_mean` | Avg distance between same-identity pairs | ↓ decreasing |
| `neg_dist_mean` | Avg distance between different-identity pairs | ↑ increasing |
| `fraction_active` | % of triplets with non-zero loss | Starts high, should fall to 10–40% |

### What good training looks like

```
Epoch 1  → loss ≈ 0.35,  active ≈ 85%   (model is learning from most triplets)
Epoch 10 → loss ≈ 0.15,  active ≈ 50%   (converging)
Epoch 30 → loss ≈ 0.05,  active ≈ 15%   (well trained, only hard triplets remain)
```

`pos_dist_mean < neg_dist_mean` — this gap widening is the key sign that training is working.

### Verification metrics (evaluated on test split)

| Metric | Description |
|---|---|
| **TAR @ FAR=0.1%** | True Accept Rate at 0.1% False Accept Rate — main face verification metric |
| **AUC** | Area Under the ROC Curve |
| **Equal Error Rate (EER)** | Point where FAR = FRR — lower is better |
| **Threshold** | L2 distance cutoff used for same/different decision |

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `No GPU found, using CPU` | CPU-only PyTorch installed | `pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124` |
| `caffe2_nvrtc.dll not found` | CUDA Toolkit not installed | Install CUDA Toolkit 12.4 from NVIDIA, restart PC |
| CUDA out of memory | Batch too large for 2 GB VRAM | `python train.py --batch_size 2` |
| `splits/train.txt not found` | split_dataset.py not run yet | `python split_dataset.py` |
| `pin_memory` UserWarning | Running on CPU | Harmless warning, ignore or use `--workers 0` |
