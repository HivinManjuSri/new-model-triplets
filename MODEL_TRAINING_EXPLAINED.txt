1. OVERVIEW
================================================================================

This is a SIAMESE TRIPLET NETWORK for face verification/recognition.

Goal: Learn to embed face images into a 128-dimensional space where:
  - Same person's faces are CLOSE together
  - Different people's faces are FAR apart

Method: Triplet learning - the model learns by comparing 3 images at once:
  - Anchor (A): A face of person X
  - Positive (P): Different photo of the same person X
  - Negative (N): A face of a different person Y

2. DATA INPUT (TYPE, SIZE, AND WHY)
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│ DATASET STRUCTURE                                                       │
└─────────────────────────────────────────────────────────────────────────┘

Total Dataset: 10,104 pre-built triplets
  - Training:   8,083 triplets (80%)
  - Validation: 1,010 triplets (10%)
  - Test:       1,011 triplets (10%)

Each triplet folder contains:
  - A_XXXXX.ppm  (Anchor image - person XXXXX)
  - P_XXXXX.ppm  (Positive image - same person XXXXX, different photo)
  - N_YYYYY.ppm  (Negative image - different person YYYYY)

Format: PPM image files (Portable Pixmap)


┌─────────────────────────────────────────────────────────────────────────┐
│ IMAGE PROCESSING PIPELINE                                               │
└─────────────────────────────────────────────────────────────────────────┘

INPUT SIZE: 112 × 112 pixels (RGB)
WHY 112×112?
  ✓ Large enough to preserve facial features
  ✓ Small enough for efficient GPU processing
  ✓ Standard size for face recognition models

PREPROCESSING STEPS:
  1. Load PPM file → Convert to RGB PIL Image
  2. Resize to 112×112 pixels
  3. Apply data augmentation (TRAINING ONLY):
     - Random horizontal flip (50% chance)
     - Color jitter (brightness±20%, contrast±20%, saturation±10%, hue±5%)
     - Random grayscale conversion (2% chance)
  4. Convert to tensor: (3, 112, 112) float32
  5. Normalize using ImageNet statistics:
     - Mean: [0.485, 0.456, 0.406]
     - Std:  [0.229, 0.224, 0.225]

WHY IMAGENET NORMALIZATION?
  → The ResNet-50 backbone was pretrained on ImageNet
  → Using same normalization ensures the pretrained weights work properly


┌─────────────────────────────────────────────────────────────────────────┐
│ BATCH STRUCTURE                                                         │
└─────────────────────────────────────────────────────────────────────────┘

Batch Size: Auto-selected based on GPU VRAM
  - 20+ GB VRAM → 64 triplets per batch
  - 10-20 GB   → 32 triplets per batch
  - 6-10 GB    → 16 triplets per batch
  - 4-6 GB     → 8 triplets per batch
  - <4 GB      → 4 triplets per batch
  - CPU only   → 8 triplets per batch

Each batch contains:
  - anchor_batch:   (batch_size, 3, 112, 112) - float32
  - positive_batch: (batch_size, 3, 112, 112) - float32
  - negative_batch: (batch_size, 3, 112, 112) - float32

WHY THIS STRUCTURE?
  → Triplet loss requires all 3 images together to compute
  → Batch processing enables parallel GPU computation
  → Larger batches = more stable gradients, but need more VRAM

3. MODEL ARCHITECTURE
================================================================================

                    ┌────────────────────────────┐
                    │    INPUT IMAGE             │
                    │    (3, 112, 112)           │
                    └────────────┬───────────────┘
                                 │
                    ┌────────────▼───────────────┐
                    │    ResNet-50 BACKBONE      │
                    │    (Pretrained on ImageNet)│
                    │                            │
                    │  • 50 convolutional layers │
                    │  • Residual connections    │
                    │  • Batch normalization     │
                    │  • Original FC removed     │
                    └────────────┬───────────────┘
                                 │
                        2048-d feature vector
                                 │
                    ┌────────────▼───────────────┐
                    │   EMBEDDING HEAD           │
                    │                            │
                    │  Linear(2048 → 512)        │
                    │  BatchNorm1d(512)          │
                    │  ReLU                      │
                    │  Dropout(p=0.3)            │
                    │  Linear(512 → 128)         │
                    └────────────┬───────────────┘
                                 │
                         128-d embedding
                                 │
                    ┌────────────▼───────────────┐
                    │   L2 NORMALIZATION         │
                    │   (Map to unit sphere)     │
                    └────────────┬───────────────┘
                                 │
                    ┌────────────▼───────────────┐
                    │   OUTPUT EMBEDDING         │
                    │   (128,) - L2 norm = 1.0   │
                    └────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────┐
│ KEY ARCHITECTURAL CHOICES                                               │
└─────────────────────────────────────────────────────────────────────────┘

1. ResNet-50 Backbone (PRETRAINED)
   WHY? Transfer learning from ImageNet gives better feature extraction
   - Pretrained weights already understand edges, textures, shapes
   - Faster convergence than training from scratch
   - Better performance with limited face data

2. Embedding Dimension: 128
   WHY 128?
   - Large enough to capture facial identity information
   - Small enough to be memory-efficient
   - Industry standard (FaceNet uses 128, ArcFace uses 512)

3. L2 Normalization (Unit Sphere)
   WHY?
   - Constrains embeddings to unit hypersphere
   - Makes distance comparisons consistent
   - Euclidean distance becomes equivalent to angular distance
   - Prevents magnitude from affecting similarity

4. Dropout (p=0.3) 
   WHY?
   - Prevents overfitting during training
   - Forces network to learn robust features
   - Turned off during validation/testing


┌─────────────────────────────────────────────────────────────────────────┐
│ SIAMESE ARCHITECTURE (WEIGHT SHARING)                                   │
└─────────────────────────────────────────────────────────────────────────┘

The SAME network is used for all 3 branches:

    Anchor ──┐
             ├──→ SHARED MODEL (same weights) → embeddings
    Positive─┤
             │
    Negative─┘

This is called "Siamese" architecture - ensures:
  → Consistent embedding space for all images
  → Same features extracted regardless of input order
  → 3× memory efficiency vs separate networks

4. TRAINING PIPELINE - STEP BY STEP
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│ STAGE 1: INITIALIZATION                                                 │
└─────────────────────────────────────────────────────────────────────────┘

1. Load Configuration
   - Epochs: 30 (default)
   - Learning rate: 0.0001
   - Margin: 0.2 (triplet loss margin)
   - Embedding dimension: 128
   - Batch size: Auto-detected based on GPU

2. Initialize Datasets
   - Load training split: 8,083 triplets
   - Load validation split: 1,010 triplets
   - Create DataLoaders (shuffle=True for training)
   - Setup data augmentation (training only)
   - Worker threads: 4 (parallel data loading)

3. Build Model
   - Create ResNet-50 backbone
   - Load ImageNet pretrained weights
   - Replace final FC with embedding head
   - Move model to GPU (if available)

4. Setup Optimization
   - Optimizer: Adam (adaptive learning rate)
     * Learning rate: 0.0001
     * Weight decay: 0.00001 (L2 regularization)
   - Loss: TripletLoss (margin = 0.2)
   - Scheduler: CosineAnnealingLR
     * Smoothly reduces LR from 0.0001 → 0.000001 over 30 epochs

5. Initialize Tracking
   - best_val_loss = infinity
   - training_log = []
   - start_epoch = 1


┌─────────────────────────────────────────────────────────────────────────┐
│ STAGE 2: SINGLE BATCH FORWARD PASS (What happens to ONE batch)         │
└─────────────────────────────────────────────────────────────────────────┘

Let's say batch_size = 32:

STEP 1: Load Data
  ────────────────────────────────────────────────────────────
  DataLoader fetches 32 triplets from disk
  
  anchor_batch:   (32, 3, 112, 112) - 32 anchor images
  positive_batch: (32, 3, 112, 112) - 32 positive images
  negative_batch: (32, 3, 112, 112) - 32 negative images

STEP 2: Move to GPU
  ────────────────────────────────────────────────────────────
  All 3 batches transferred to GPU memory
  (If no GPU, stays on CPU - slower)

STEP 3: Forward Pass Through Model
  ────────────────────────────────────────────────────────────
  ea = model(anchor_batch)     → (32, 128) embeddings
  ep = model(positive_batch)   → (32, 128) embeddings
  en = model(negative_batch)   → (32, 128) embeddings
  
  Each embedding is L2-normalized (norm = 1.0)

STEP 4: Compute Triplet Loss
  ────────────────────────────────────────────────────────────
  For each triplet i in the batch:
  
    d_pos[i] = ||ea[i] - ep[i]||²  (distance: anchor to positive)
    d_neg[i] = ||ea[i] - en[i]||²  (distance: anchor to negative)
    
    loss[i] = max(0, d_pos[i] - d_neg[i] + 0.2)
    
  Desired outcome:
    ✓ d_pos should be SMALL (same person = close)
    ✓ d_neg should be LARGE (different person = far)
    ✓ Margin 0.2 enforces separation: d_neg > d_pos + 0.2
  
  Batch loss = mean(loss[0], loss[1], ..., loss[31])
  
  Statistics tracked:
    • Average positive distance
    • Average negative distance
    • Fraction of "active" triplets (loss > 0)

STEP 5: Backward Pass (TRAINING ONLY)
  ────────────────────────────────────────────────────────────
  optimizer.zero_grad()              # Clear old gradients
  loss.backward()                    # Compute gradients
  clip_grad_norm_(max_norm=1.0)     # Prevent exploding gradients
  optimizer.step()                   # Update model weights

STEP 6: Update Weights
  ────────────────────────────────────────────────────────────
  All parameters (ResNet-50 + embedding head) updated by:
    θ_new = θ_old - learning_rate × gradient
  
  Adam optimizer uses momentum and adaptive rates per parameter


┌─────────────────────────────────────────────────────────────────────────┐
│ STAGE 3: COMPLETE EPOCH (Processing all batches)                       │
└─────────────────────────────────────────────────────────────────────────┘

Training Epoch:
  ────────────────────────────────────────────────────────────
  Total batches = 8,083 / batch_size
  
  For batch_size=32: ~253 batches per epoch
  
  FOR each batch in training_loader:
    1. Load anchor, positive, negative
    2. Forward pass → get embeddings
    3. Compute loss
    4. Backward pass → compute gradients
    5. Update weights
    6. Track statistics
    7. Print progress every 50 batches
  
  END FOR
  
  Calculate epoch average:
    - Average loss
    - Average positive distance
    - Average negative distance
    - Fraction of active triplets

Validation Epoch:
  ────────────────────────────────────────────────────────────
  Total batches = 1,010 / batch_size
  
  Set model.eval()  ← Disables dropout, batch norm in eval mode
  
  WITH torch.no_grad():  ← No gradient computation (faster)
    FOR each batch in validation_loader:
      1. Load anchor, positive, negative
      2. Forward pass → get embeddings
      3. Compute loss (NO backward pass)
      4. Track statistics
    END FOR
  
  Calculate epoch average (same metrics as training)


┌─────────────────────────────────────────────────────────────────────────┐
│ STAGE 4: LEARNING RATE SCHEDULING                                      │
└─────────────────────────────────────────────────────────────────────────┘

After each epoch, the learning rate is adjusted:

CosineAnnealingLR schedule over 30 epochs:

  Epoch 1:  LR = 0.00010000  ──╮
  Epoch 2:  LR = 0.00009945    │
  Epoch 3:  LR = 0.00009781    │
  Epoch 5:  LR = 0.00009330    │  Smooth
  Epoch 10: LR = 0.00007500    │  Cosine
  Epoch 15: LR = 0.00005000    ├─ Decay
  Epoch 20: LR = 0.00002500    │
  Epoch 25: LR = 0.00000670    │
  Epoch 28: LR = 0.00000219    │
  Epoch 30: LR = 0.00000100  ──╯

WHY Cosine Annealing?
  → Fast learning early (large steps)
  → Fine-tuning later (small steps)
  → Smooth decay prevents sudden jumps
  → Better convergence to local minimum


┌─────────────────────────────────────────────────────────────────────────┐
│ STAGE 5: CHECKPOINTING & LOGGING                                       │
└─────────────────────────────────────────────────────────────────────────┘

After each epoch:

1. Save "last_model.pt"
   - Contains: model weights, optimizer state, scheduler state
   - Purpose: Resume training if interrupted

2. Check if validation loss improved
   IF val_loss < best_val_loss:
     best_val_loss = val_loss
     Save "best_model.pt"  ← This is the model you'll use later!
     Print "*** New best ***"

3. Append to training_log.json
   {
     "epoch": 5,
     "lr": 0.00009330,
     "train": {
       "loss": 0.1234,
       "pos_dist_mean": 0.4567,
       "neg_dist_mean": 0.8901,
       "fraction_active": 0.65
     },
     "val": {
       "loss": 0.1567,
       ...
     },
     "elapsed": 45.2
   }

5. LOSS FUNCTION DETAILS
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│ TRIPLET LOSS FORMULA                                                    │
└─────────────────────────────────────────────────────────────────────────┘

For a single triplet (Anchor, Positive, Negative):

  L = max(0, ||A - P||² - ||A - N||² + margin)

Where:
  ||A - P||²  = squared distance between anchor and positive (should be SMALL)
  ||A - N||²  = squared distance between anchor and negative (should be LARGE)
  margin      = 0.2 (minimum required separation)

INTERPRETATION:
  
  ✓ If d(A,N) > d(A,P) + 0.2  →  Loss = 0 (good triplet, already separated)
  ✗ If d(A,N) ≤ d(A,P) + 0.2  →  Loss > 0 (violation, needs learning)


┌─────────────────────────────────────────────────────────────────────────┐
│ WHAT THE LOSS OPTIMIZES                                                 │
└─────────────────────────────────────────────────────────────────────────┘

The gradient descent pushes the model to:

1. PULL same-person faces closer together
   → Minimize d(Anchor, Positive)

2. PUSH different-person faces farther apart
   → Maximize d(Anchor, Negative)

3. Maintain margin of 0.2 between them
   → Ensure clear decision boundary

Over training:
  
  Epoch 1:   d_pos ≈ 0.8,  d_neg ≈ 0.9   (barely separated)
  Epoch 10:  d_pos ≈ 0.5,  d_neg ≈ 1.1   (better separation)
  Epoch 30:  d_pos ≈ 0.2,  d_neg ≈ 1.4   (well separated)


┌─────────────────────────────────────────────────────────────────────────┐
│ FRACTION ACTIVE (Important Metric)                                     │
└─────────────────────────────────────────────────────────────────────────┘

Fraction Active = percentage of triplets with loss > 0

  Early training:  ~90% active  (most triplets violating margin)
  Mid training:    ~50% active  (half the triplets learned)
  Late training:   ~20% active  (most triplets already correct)

When fraction_active → 0:
  → Model has learned to separate nearly all triplets
  → Further training yields diminishing returns

6. COMPLETE TRAINING FLOW (EPOCH BY EPOCH)
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│ EPOCH 1                                                                 │
└─────────────────────────────────────────────────────────────────────────┘

1. Set LR = 0.0001
2. model.train() ← Enable dropout, batch norm in training mode
3. Shuffle training data
4. FOR each of ~253 training batches:
     - Load 32 triplets from disk
     - Apply random augmentation
     - Transfer to GPU
     - Forward pass: anchor, positive, negative → embeddings
     - Compute triplet loss
     - Backward pass: compute gradients
     - Clip gradients (prevent explosion)
     - Update all model parameters
     - Every 50 batches: print progress
5. Calculate training statistics (average over all batches)
6. Switch to validation mode
7. model.eval() ← Disable dropout
8. FOR each of ~32 validation batches:
     - Load 32 triplets (no augmentation)
     - Forward pass only (no gradients)
     - Compute loss (for monitoring only)
9. Calculate validation statistics
10. Print epoch summary:
      [Train] loss=0.2345  d+=0.7123  d-=0.8456  active=87%
      [Val  ] loss=0.2567  d+=0.7234  d-=0.8345  active=85%
      Epoch time: 120.5s
11. Save checkpoints
12. Update learning rate schedule
13. Log results to JSON

┌─────────────────────────────────────────────────────────────────────────┐
│ EPOCHS 2-29                                                             │
└─────────────────────────────────────────────────────────────────────────┘

Repeat the same process, but:
  - Learning rate gradually decreases (cosine schedule)
  - Loss generally decreases (model improving)
  - d_pos decreases (same faces getting closer)
  - d_neg increases (different faces getting farther)
  - fraction_active decreases (fewer hard triplets)
  - best_model.pt updated whenever val_loss improves

┌─────────────────────────────────────────────────────────────────────────┐
│ EPOCH 30 (FINAL)                                                        │
└─────────────────────────────────────────────────────────────────────────┘

1. LR = 0.000001 (very small, fine-tuning)
2. Same training/validation process
3. Final checkpoints saved
4. Print: "Training complete. Best val loss: 0.0834"
5. Model ready for deployment!

┌─────────────────────────────────────────────────────────────────────────┐
│ WHAT HAPPENS AFTER TRAINING                                            │
└─────────────────────────────────────────────────────────────────────────┘

The trained model (best_model.pt) can now:

1. Embed any face image → 128-d vector
2. Compare faces by computing distance between embeddings
3. Verify if two faces are the same person:
   
   IF distance(embed1, embed2) < threshold:
     → SAME PERSON
   ELSE:
     → DIFFERENT PERSON

Typical threshold: 0.6 - 0.8 (tuned on validation set)
